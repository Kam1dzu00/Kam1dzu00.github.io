CASE-001 · В РАБОЧЕЙ БАЗЕ

Сверка исходящих УПД с системой «Честный знак»

Фоновая проверка документов и владельцев кодов через True API.

PRODUCTION

Бизнес-проблема

Статус ЭДО в бухгалтерской базе не всегда давал достаточную уверенность. Сотрудники дополнительно проверяли документы и коды маркировки вручную, что занимало до нескольких рабочих дней.

Контекст

Отдел ориентировался на статусы ЭДО в бухгалтерской базе, но для спорных документов сотрудникам приходилось отдельно заходить в систему маркировки и проверять идентификаторы и коды.

Решение

Разработал обработку для БП 3.0 с выбором организации и периода. Она находит реализации и УПД, требующие проверки, обращается к True API по идентификатору документа, извлекает коды из УПД и проверяет текущего владельца каждого кода. Массовая проверка выполняется фоновыми заданиями с контролем статуса и понятным итоговым протоколом.

Сложности

Связать реализации, счета-фактуры, электронные документы и сообщения ЭДО; корректно получить токен; обработать разные ответы True API; не потерять проблемные документы при массовой проверке.

Результат

Пользователь получает единую таблицу с реализациями, УПД, статусами, идентификаторами и количеством кодов, а также отдельную выгрузку TXT. Многодневная ручная сверка заменена одним запуском и анализом готового результата.

Влияние на бизнес

Сотрудник анализирует готовый протокол вместо ручного обхода нескольких систем. Проверка стала повторяемой, контролируемой и пригодной для регулярного использования.

02Документы ЭДО
03HTTP / True API

05Таблица результата

CASE-002 · В РАБОЧЕЙ БАЗЕ

Загрузка DataMatrix из мобильного приложения

Автоматический приём JSON с FTP и сопоставление с объектами 1С.

PRODUCTION

Бизнес-проблема

Контекст

Торговые агенты сканируют торговую точку и маркированную продукцию в мобильном приложении. Эти данные нужно было автоматически перенести в учётный контур и сопоставить со справочниками 1С.

Решение

Настроил получение JSON-файлов с FTP, парсинг даты загрузки и продажи, агента, торговой точки и DataMatrix-кодов. Добавил сопоставление с пользователями, партнёрами, номенклатурой и маршрутами, собственный регистр сведений, контроль дублей и журналирование ошибок.

CASE-003 · В РАБОЧЕЙ БАЗЕ

Влияние на бизнес

Контроль стал адресным: руководитель видит конкретное подразделение, агента и отклонение, а не общий массив технических кодов.

Контекст

Операционные задачи с маркировкой часто требовали быстро обработать несколько TXT, CSV или XML-файлов, не устанавливая дополнительное ПО и не отправляя данные во внешние сервисы.

Сохранить DataMatrix без изменения символов, обрабатывать большие наборы в браузере, показывать статистику и ошибки и формировать корректный файл результата.

Влияние на бизнес
